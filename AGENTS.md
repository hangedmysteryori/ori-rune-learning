# AGENTS.md

本專案是「世界樹觀測者 Ori」的盧恩符文靜態網站。

網站定位為：盧恩符文學習入口＋占卜服務品牌官網。

網站技術架構為：純 HTML / CSS / JavaScript，部署於 Vercel。

## 基本規則

1. 每次任務只修改必要檔案。
2. 不要主動重構全站。
3. 不要新增套件、框架、build tool 或 package.json，除非使用者明確要求。
4. 不要刪除既有內容，除非使用者明確要求。
5. 修改前先確認任務類型與允許修改範圍。
6. 若任務是檢查或驗收，禁止修改檔案。
7. 完成後必須列出修改檔案與測試方式。
8. 若使用者已指定允許修改的檔案，不得修改範圍外檔案。
9. 若需要修改範圍外檔案，必須先回報原因，不要直接修改。
10. 優先維持既有網站架構、命名、視覺風格與路由規則。
11. 不要為了小任務掃描或改動整個專案。
12. 內部連結優先使用相對路徑，例如 `/runes`、`/daily`，不要硬寫完整網址，除非是 canonical、og:url、sitemap 或外部連結需求。
13. 若修改 SEO 正式網址，需同步注意 canonical、og:url、og:image、sitemap.xml、robots.txt 與 app.js 中的 SITE_ORIGIN。
14. 若新增圖片，圖片應放在 assets/，並補上清楚的 alt 文字。
15. 若新增文章頁，需同時考慮文章頁、文章列表、內部連結、sitemap 與 Vercel rewrite。

## 盧恩內容規則

本網站中的盧恩符文內容不得使用塔羅式「正位／逆位」概念。

請統一使用以下說法：

- 正向／明亮解讀
- 負向／陰影解讀
- 明亮面
- 陰影面
- 正向狀態
- 失衡狀態

禁止使用：

- 正位
- 逆位
- 正逆位

盧恩符文在本網站系統中不以牌面方向判讀。

每一枚符文都應以「正向／明亮解讀」與「負向／陰影解讀」呈現，說明同一股力量在平衡與失衡時可能展現出的不同樣貌。

負向／陰影解讀不代表絕對壞事，而是提醒使用者看見課題、失衡、阻礙與需要調整的地方。
